Une application mise en place en partenariat avec l'entreprise ST Microelectronics par des étudiants de l'INSA de Rennes.

Découvrez
L'application peut analyser les données fournies par le capteur GPS d'un terminal Android Mais elle peut aussi analyser les données provenant d'une puce GPS externe, reliée par Bluetooth Elle peut aussi lire des informations GPS pré-enregistrées
ST-TRACS est une application Android permettant d'analyser les données fournies par une puce GPS. Elle permet, via 6 écrans principaux, de visualiser une grande variété d'informations : satellites utilisés, position, vitesse, altitude, etc. Elle permet par ailleurs de visualiser les informations de Dead Reckoning spécifiques à STMicroelectronics. L'application sera utilisée par l'entreprise pour conduire des tests grandeur nature sur cette technologie.
Le Dead Reckoning est une technologie offrant la possibilité de poursuivre la navigation assistée par GPS, même lors de la perte des signaux satellites. Ce phénomène peut survenir régulièrement dans les zones urbaines, les signaux étant facilement obstrués par les gratte-ciels, les tunnels, les parkings, etc. Le Dead Reckoning est mis en œuvre grâce à l'utilisation de capteurs (accéléromètre, gyroscope, boussole), qui permettent d'estimer les déplacements de l'utilisateur après la perte de signal.

Menu d'options

Menu d'options

Lorsqu’on clique sur le bouton settings qui se trouve dans le coin supérieur droit de chaque vue, un menu s’affichera. Il s’agit de ce que l’on appelle le menu d’options. Il comporte plusieurs champs comme nous pouvons le voir ci-contre.

Select Device

Si la tablette détecte des appareils connectés en bluetooth, l'application doit permettre à l’utilisateur de s’y connecter. Dans le cas contraire, elle va se connecter automatiquement au GPS interne. Cette vue nous propose tous les appareils (GPS) , qu’ils soient internes ou externes, ainsi que leur adresse MAC. Une fois l’appareil sélectionné, une boîte de dialogue apparaîtra, dans laquelle il y aura une barre de progression pour indiquer les étapes de connexion. Aussi, il aura la possibilité d’annuler la connexion en cours.

Playback

Cette vue nous permet de rejouer un parcours déjà enregistré. Elle contient donc tous les dossiers, ainsi que les fichiers NMEA ou binaire ST présent sur la carte SD. Lorsque l’utilisateur choisit un de ces fichiers, une boîte de dialogue s’affichera où il pourra régler plusieurs paramètres comme la vitesse à laquelle il veut que la vidéo se déroule. De plus, il pourra choisir à partir de quel moment il veut rejouer la vidéo grâce à un curseur de contrôle. Après avoir choisi les réglages qui lui conviennent, l’utilisateur pourra lancer le playback ou annuler en cliquant sur les deux boutons respectifs.

Upload File

Cette vue permet d’envoyer des fichiers présents sur la carte SD vers un serveur FTP si la connexion réseau est établie (3G/wifi). L’utilisateur a la possiblité de choisir le fichier qu’il veut enregistrer et une fois qu’il l’a sélectionné une boîte de dialogue apparaît qui lui spécifiera le nom du fichier choisi, la progression de l’enregistrement et qui lui donnera l’opportunité d’annuler l’envoi. De plus, l’utilisateur y trouvera à titre informatif l’adresse IP du serveur FTP, le port de connexion, ainsi que le login et son mot de passe.

Settings

Cette vue permet à l’utilisateur de faire plusieurs réglages et ressemblera à la figure ci-contre. Les différents réglages possibles seront :

  • l’Écran d’accueil au démarrage de l’application ;
  • l’orientation de l’écran (auto, paysage, portrait) ;
  • l’activation ou la désactivation de l’extinction automatique de l’écran ;
  • le type de carte ;
  • le datum (WGS84 (World Geodetic System 1984 ) par défaut) ;
  • le format de l’heure (24H, AM/PM pour la version en anglais) ;
  • le format des coordonnées GNSS (DD°MM.MMMM’; DD°MM’SS.SSS) ;
  • l’unité des vitesses et distances (Métrique, Nautique, Impérial) ;
  • l’unité d’altitude (Métrique, Impérial).
