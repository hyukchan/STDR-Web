ST TRACS

Teseo Receiver Advanced Control Suite

Une application Android dédiée à l'analyse de données GPS & Dead Reckoning

Découvrir

ST-TRACS est une application développée par 8 étudiants du département informatique de l'INSA de Rennes, en collaboration avec STMicroelectronics

Le client

La division automobile de STMicroelectronics nous a sollicité pour répondre à un besoin réel de l'entreprise

Son besoin

L'entreprise veut tester sa solution de Dead Reckoning, une technologie offrant la possibilité de poursuivre la navigation assistée par GPS même lors de la perte des signaux satellites.

La solution

La solution envisagée est une application Android permettant aux collaborateurs de STMicroelectronics de visualiser les données GPS et Dead Reckoning renvoyées par la puce testée.

Présentation

Skyview

  • Visualisation des satellites visibles
  • Orientation
  • Comparaison du positionnement via GPS et via le dead reckoning
  • Graphes de comparaison des 2 types de positionnement
  • Informations sur le dead reckoning
  • Informations précises sur chaque satellite capté
  • Satellites classés par qualité du signal (CN0)
  • Vue GPS classique
  • Précision du positionnement GPS (HDOP, PDOP)
  • Visualisation des données reçues

Extras

Select Device

Si la tablette détecte des appareils connectés en bluetooth, l'application doit permettre à l’utilisateur de s’y connecter. Dans le cas contraire, elle se connecte automatiquement au GPS interne. Cette vue nous propose tous les appareils (GPS) , qu’ils soient internes ou externes, ainsi que leur adresse MAC. Une fois l’appareil sélectionné, une boîte de dialogue apparaît, dans laquelle il y a une barre de progression pour indiquer les étapes de connexion ainsi qu'un bouton permettant d’annuler cette connexion en cours.

Playback

Cette vue nous permet de rejouer un parcours déjà enregistré. Pour cela elle donne accès à un explorateur de fichier où l’utilisateur pourra selectionner le fichier qu'il veut rejouer. Après la séléction, une boîte de dialogue s’affiche où il peut régler plusieurs paramètres comme la vitesse à laquelle il veut que la vidéo se déroule. De plus, il peut choisir à partir de quel moment il veut rejouer la vidéo grâce à un curseur de contrôle. Après avoir choisi les réglages qui lui conviennent, l’utilisateur peut lancer le playback ou annuler en cliquant sur les deux boutons respectifs.

Upload File

Cette vue permet d’envoyer des fichiers présents sur la carte SD vers un serveur FTP si la connexion réseau est établie (3G/wifi). L’utilisateur a la possiblité de choisir le fichier qu’il veut enregistrer. Une fois qu’il l’a sélectionné une boîte de dialogue apparaît avec la possibilité de spécifier le nom du fichier choisi, la progression de l’enregistrement ainsi ainsi qu'un bouton pour annuler l’envoi. La vue présente aussi à titre informatif l’adresse IP du serveur FTP, le port de connexion, ainsi que le login et son mot de passe.

Settings

Cette vue permet à l’utilisateur de faire plusieurs réglages et ressemble à la figure ci-contre. Les différents réglages possibles seront :

  • l’Écran d’accueil au démarrage de l’application ;
  • l’orientation de l’écran (auto, paysage, portrait) ;
  • l’activation ou la désactivation de l’extinction automatique de l’écran ;
  • le type de carte ;
  • le datum (WGS84 (World Geodetic System 1984 ) par défaut) ;
  • le format de l’heure (24H, AM/PM pour la version en anglais) ;
  • le format des coordonnées GNSS (DD°MM.MMMM’; DD°MM’SS.SSS) ;
  • l’unité des vitesses et distances (Métrique, Nautique, Impérial) ;
  • l’unité d’altitude (Métrique, Impérial).
